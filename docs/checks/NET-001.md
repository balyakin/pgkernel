# NET-001 — TCP Keepalive

## Why it matters

TCP keepalive settings affect how quickly dead remote connections are detected.

## Detection

- `net.ipv4.tcp_keepalive_time`
- `net.ipv4.tcp_keepalive_intvl`
- `net.ipv4.tcp_keepalive_probes`

## Status logic

- `tcp_keepalive_time <= 600` -> pass
- higher values -> info recommendation

## Fix

```bash
sysctl -w net.ipv4.tcp_keepalive_time=600
```

Safety: `safe-runtime`

Reference: https://www.postgresql.org/docs/current/runtime-config-connection.html
