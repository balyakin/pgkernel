# MEM-005 — OOM Killer Score for PostgreSQL

## Why it matters

Without OOM protection, PostgreSQL may be killed first during host-level memory pressure.

## Detection

- PostgreSQL PID from `postmaster.pid` or `pgrep`
- `/proc/<pid>/oom_score_adj`

## Status logic

- `-1000` -> pass
- `<0` -> info
- `>=0` -> warn

## Fix

```bash
PG_PID=$(head -1 /var/lib/postgresql/*/main/postmaster.pid 2>/dev/null || pgrep -o postgres)
echo -1000 > /proc/$PG_PID/oom_score_adj
```

Safety: `safe-runtime`

Reference: https://www.postgresql.org/docs/current/kernel-resources.html
