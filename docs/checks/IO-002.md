# IO-002 — Dirty Ratios

## Why it matters

High dirty page limits can create bursty flush behavior and stall foreground PostgreSQL writes.

## Detection

- `/proc/sys/vm/dirty_background_ratio`
- `/proc/sys/vm/dirty_ratio`

## Status logic

- `dirty_background_ratio <= 5` and `dirty_ratio <= 20` -> pass
- otherwise -> warn

## Fix

```bash
sysctl -w vm.dirty_background_ratio=3
sysctl -w vm.dirty_ratio=10
```

Safety: `safe-runtime`

Reference: https://www.postgresql.org/docs/current/kernel-resources.html
