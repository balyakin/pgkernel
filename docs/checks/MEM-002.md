# MEM-002 — Static Huge Pages Configuration

## Why it matters

Static huge pages reduce TLB pressure and can improve PostgreSQL throughput when `shared_buffers` is large.

## Detection

- `huge_pages` from `postgresql.conf`
- `HugePages_Total`, `HugePages_Free`, `Hugepagesize` from `/proc/meminfo`

## Status logic

- `huge_pages=off` -> warn
- `huge_pages on/try` and `HugePages_Total=0` -> crit
- allocated pages insufficient for `shared_buffers` -> warn
- enough pages -> pass

## Fix

```bash
echo 4506 > /proc/sys/vm/nr_hugepages
echo 'vm.nr_hugepages = 4506' >> /etc/sysctl.d/99-postgresql.conf
```

Safety: `high-risk`

Reference: https://www.postgresql.org/docs/current/kernel-resources.html
