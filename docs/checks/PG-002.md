# PG-002 — work_mem Sanity

## Why it matters

`work_mem` is per operation and per session. High values multiplied by `max_connections` can exceed RAM under concurrent load.

## Detection

- `work_mem` and `max_connections` from `postgresql.conf`
- total RAM from `/proc/meminfo`

## Status logic

- if `work_mem * max_connections > 50% RAM` -> warn
- otherwise -> pass

## Fix

Lower `work_mem` or reduce `max_connections` to keep worst-case memory envelope controlled.

Safety: `reboot-required`

Reference: https://www.postgresql.org/docs/current/runtime-config-resource.html
