# PG-003 — WAL Configuration

## Why it matters

WAL tuning affects checkpoint smoothness, write amplification, and transaction latency.

## Detection

- `wal_buffers`
- `max_wal_size`
- `checkpoint_completion_target`

## Status logic

- `wal_buffers = -1` or `>=16MB` -> pass
- low explicit `wal_buffers` -> warn
- `checkpoint_completion_target < 0.7` -> warn

## Fix

Set `wal_buffers=-1` and `checkpoint_completion_target=0.9` as a robust baseline for most production workloads.

Safety: `reboot-required`

Reference: https://www.postgresql.org/docs/current/wal-configuration.html
