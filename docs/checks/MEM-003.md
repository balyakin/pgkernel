# MEM-003 — vm.swappiness

## Why it matters

High swappiness can move PostgreSQL memory to swap under pressure, causing severe latency and throughput degradation.

## Detection

- `/proc/sys/vm/swappiness`

## Status logic

- `<=10` -> pass
- `11..60` -> warn
- `>60` -> crit

## Fix

```bash
sysctl -w vm.swappiness=1
echo 'vm.swappiness = 1' >> /etc/sysctl.d/99-postgresql.conf
```

Safety: `safe-runtime`

Reference: https://www.postgresql.org/docs/current/kernel-resources.html
