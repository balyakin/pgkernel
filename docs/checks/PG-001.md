# PG-001 — shared_buffers vs Total RAM

## Why it matters

`shared_buffers` too low wastes host RAM, while too high can increase memory pressure and double-buffering effects.

## Detection

- `shared_buffers` from `postgresql.conf`
- total RAM from `/proc/meminfo`

## Status logic

- `20%-35%` of RAM -> pass
- `<20%` -> warn
- `>40%` -> warn
- default `128MB` on host `>4GB` RAM -> crit

## Fix

Set `shared_buffers` to around `25%` of host RAM as a baseline, then tune using workload metrics.

Safety: `reboot-required`

Reference: https://www.postgresql.org/docs/current/runtime-config-resource.html
