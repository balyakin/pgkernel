# MEM-004 — vm.overcommit_memory

## Why it matters

Unsafe overcommit behavior can increase PostgreSQL OOM-kill risk under memory stress.

## Detection

- `/proc/sys/vm/overcommit_memory`

## Status logic

- `2` -> pass
- `0` -> warn
- `1` -> crit

## Fix

```bash
sysctl -w vm.overcommit_memory=2
echo 'vm.overcommit_memory = 2' >> /etc/sysctl.d/99-postgresql.conf
```

Safety: `safe-runtime`

Reference: https://www.postgresql.org/docs/current/kernel-resources.html
